## 🎓 Why two IDEs?

We start with **VS Code**. It's light and fast — perfect while you focus on learning syntax.

Later, we add **Android Studio** for its emulator manager and deeper Android tools. By the end, you'll be comfortable with both — just like real Flutter teams.

## ✅ By the end of this chapter, you can:

- Run `flutter doctor` and understand the result
- Create a new Flutter project
- Run your app in **Chrome**, an **Android emulator**, or a **real phone**
- Use hot reload (`r`) and hot restart (`R`)
- Explain why one app can run on a browser, a phone, and a desktop
